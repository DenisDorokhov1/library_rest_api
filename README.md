## Микросервис на Flask для управления библиотечным каталогом. 
Проект реализован в качестве полноценного REST API с использованием чистого SQL через модуль sqlite3 и ручным управлением связями между таблицами.

### Архитектура данных
В проекте реализована реляционная модель, которая состоит из:

* Authors: Хранит ФИО авторов.

* Books: Хранит названия книг и внешний ключ author_id.

Между таблицами реализована поддержка FOREIGN KEY на уровне SQLite. При удалении автора срабатывает ON DELETE CASCADE, что автоматически удаляет все его книги.


### Структура файлов в app_library
* route.py: Настройка Flask-RESTX, описание API-ресурсов и обработка HTTP-статусов.

* models.py: SQL-логика, создание таблиц, dataclasses и работа с подключением к SQLite.

* schemas.py: Описание схем Marshmallow для сериализации и логика валидации полей.



### Запуск приложения
Проект полностью контейнеризирован.

После копирования репозитория, запустить контейнер: ```docker compose up```

После успешного запуска контейнера интерактивная документация Swagger UI доступна в вашем браузере по адресу
:

http://127.0.0.1:8000/docs

### Реализованные эндпоинты:
 Авторы (/api/authors)
* GET /api/authors/ — Получить список всех авторов.

* POST /api/authors/ — Создать нового автора.

В Swagger настроена валидация: поля id скрыто для ввода, а поле middle_name является опциональным.

* GET /api/authors/<author_id> — Посмотреть все книги конкретного автора по его ID.

* DELETE /api/authors/<author_id> — Удалить автора и каскадно удалить все его книги.

Книги (/api/books)
* GET /api/books/ — Получить список всех книг.

* POST /api/books/ — Добавить новую книгу (с валидацией уникальности названия и проверки существования author_id).

* GET /api/books/<book_id> — Получить информацию о книге по ID.

* PUT /api/books/<book_id> — Полное обновление данных книги.

* PATCH /api/books/<book_id> — Частичное обновление параметров книги (только title или author_id).

* DELETE /api/books/<book_id> — Удалить книгу из каталога.
